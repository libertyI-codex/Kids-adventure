# こどもの冒険 最終設計

## 1. 設計の前提

この文書は、実装直前の最終設計です。前回設計と今回の確定条件に差がある場合は、今回の確定条件を優先します。

子ども向け表示名は **結羽ちゃんの冒険**、内部的なプロジェクト名は **こどもの冒険** です。Ver.1.0では結羽1人を対象にしますが、データには `profileId` を持たせ、将来の複数プロフィール化に備えます。

主目的は、ゲーム攻略ではなく生活習慣の定着です。失敗、罰、ランキング、他人との競争は設計に入れません。

## 2. 基本体験

Ver.1.0で完成させる中心体験は次の通りです。

```text
今日のおしごとをする
↓
スターをもらう
↓
ぬりえを解放する
↓
自由に色を塗る
↓
まほうの仕上げで作品が変身する
↓
完成した作品が森に登場する
↓
今日のぼうけんと思い出が残る
```

この流れを最優先にし、Ver.1.0では機能を増やしすぎないようにします。

## 3. 技術方針

使用技術は次に限定します。

- HTML
- CSS
- JavaScript
- localStorage
- GitHub Pages

外部API、外部AI、有料サービス、ログイン、クラウド保存、外部画像URL、外部フォントCDN、外部アイコンCDNは使いません。

Ver.1.0ではPWAとService Workerは実装対象外です。ただし、将来PWA化しやすいよう、外部依存を持たないファイル構成にします。

## 4. 画面一覧

Ver.1.0の実装対象画面は次の11画面です。

| 画面 | 主な利用者 | 役割 |
|---|---|---|
| ホーム | 子ども | 今日の入口。スター、次の行動、森への導線を見せる |
| 今日のおしごと | 子ども | はみがき等を完了し、スターを獲得する |
| スター獲得演出 | 子ども | 達成感を短く表示する |
| ぬりえ一覧 | 子ども | 解放済み・未解放のぬりえを選ぶ |
| ぬりえ編集 | 子ども | SVGぬりえに色を塗る |
| まほうの仕上げ | 子ども | AI風ではなく「まほう」として作品変身を演出する |
| 森 | 子ども | 完成作品が登場する思い出の森を見る |
| 今日のぼうけん | 子ども・親 | 今日の達成、スター、作品を締めとして見る |
| 今日の作品・簡易アルバム | 子ども・親 | 完成作品を見返す |
| 親モード | 親 | おしごと、スター、名前、ひとことを管理する |
| データ管理 | 親 | JSONバックアップ、復元、初期化、バージョン確認を行う |

## 5. 画面遷移

基本導線は子どもが迷わないように浅くします。

```text
ホーム
├─ 今日のおしごと
│  └─ スター獲得演出
│     ├─ 今日のおしごと
│     ├─ ぬりえ一覧
│     └─ 今日のぼうけん
├─ ぬりえ一覧
│  └─ ぬりえ編集
│     └─ まほうの仕上げ
│        └─ 森
├─ 森
│  ├─ 作品詳細
│  └─ 今日の作品・簡易アルバム
├─ 今日のぼうけん
└─ 保護者ボタン長押し
   └─ 確認画面
      └─ 親モード
         └─ データ管理
```

下部ナビゲーション候補は次の5つです。

- ホーム
- おしごと
- ぬりえ
- もり
- さくひん

親モードは下部ナビに入れず、ホーム右上の保護者ボタンから入ります。

## 6. スター設計

スターは2種類に分けます。

| 種類 | 減るか | 用途 |
|---|---:|---|
| 累計スター | 減らない | 成長記録、森レベル、将来の解放条件 |
| つかえるスター | 減る | ぬりえ解放に使う残高 |

おしごと完了時は、累計スターとつかえるスターの両方に加算します。ぬりえ解放時は、つかえるスターだけを減らします。

子ども向け画面では、混乱を避けるために表示の優先度を分けます。

- ホーム: 「つかえるほし」を大きく表示し、「これまでのほし」を小さく表示
- 今日のおしごと: 「きょうあつめたほし」を表示
- ぬりえ一覧: 「ぬりえにつかえるほし」を表示
- 親モード: 累計スター、つかえるスター、スター履歴を明確に表示

## 7. 今日のおしごと設計

Ver.1.0の初期おしごとは次の4つです。

| ID | 表示名 | 初期報酬 | 初期状態 |
|---|---|---:|---|
| `task_brush_teeth` | はみがき | 1 | 有効 |
| `task_get_dressed` | おきがえ | 1 | 有効 |
| `task_picture_book` | えほん | 1 | 有効 |
| `task_watering` | おみずあげ | 1 | 有効 |

1つのおしごとは1日1回だけスター獲得可能です。二重加算は、日別記録内の完了済み `taskId` とスター履歴の `sourceId` で防止します。

誤操作の取り消しは子ども画面からは簡単に行えません。Ver.1.0では親モードから訂正し、スター履歴に訂正記録を残します。

## 8. ぬりえ設計

ぬりえはSVGを基本にします。SVG内を複数の塗り領域に分け、子どもは色パレットから色を選んで領域をタップします。

Ver.1.0の初期ぬりえは次の4種類です。

| 解放順 | ID | 表示名 | 必要スター | 森での分類 |
|---:|---|---|---:|---|
| 1 | `coloring_butterfly_001` | ちょうちょ | 1 | 生き物 |
| 2 | `coloring_flower_001` | おはな | 2 | 植物 |
| 3 | `coloring_rabbit_001` | うさぎ | 3 | 生き物 |
| 4 | `coloring_cat_001` | ねこ | 4 | 生き物 |

Ver.1.0の色パレットは10色を推奨します。

- あか
- ぴんく
- きいろ
- おれんじ
- みどり
- みずいろ
- あお
- むらさき
- ちゃいろ
- しろ

完成条件は「全領域塗り必須」にしません。少なくとも1領域以上塗られていれば完成できます。塗った領域数は、森でのサイズや演出に反映します。

作品はBase64画像を大量保存しません。保存するのはテンプレートID、領域別カラー、使用色、代表色、完成日、配置情報です。再表示時はSVGテンプレートと保存色から再描画します。

## 9. まほうの仕上げ設計

子ども向け画面では「AI」という言葉を使わず、**まほうの仕上げ** と表現します。

流れ:

```text
ぬりえ完成
↓
まほうをかける
↓
色を調べているような演出
↓
陰影・光沢・輪郭・背景効果を追加
↓
作品が森へ飛んでいく
↓
森に登場
```

演出は5秒以内、スキップ可能にします。入力結果の反映は次のルールで行います。

| 条件 | 森への反映 |
|---|---|
| 明るい色が多い | 明るい場所へ配置 |
| 青系が多い | 水辺または空側へ配置 |
| 緑系が多い | 木や草の近くへ配置 |
| ピンク・赤系が多い | 花の近くへ配置 |
| 塗った領域が多い | 少し大きく表示 |
| 同じ種類を複数完成 | 仲間として追加表示 |

ランダム要素を使う場合は、作品IDから生成した固定シードを使います。同じ作品を開くたびに見た目や配置が変わらないようにします。

## 10. 森・ワールド設計

内部的には森専用アプリにせず、森を最初のワールドとして扱います。

```text
worlds
└─ world_forest
```

将来追加できる世界:

- 海
- 空
- 町
- どうぶつ村
- 星の島

Ver.1.0では森だけを実装対象にします。

## 11. SVG中心の森が適切か

Ver.1.0では、森画面はSVG中心のレイヤー構造が適切です。

理由:

- 作品を個別要素として配置しやすい
- タップ判定を作品単位で持ちやすい
- 色を反映したSVG再描画と相性がよい
- DOMで作品詳細ポップアップを出しやすい
- Canvasだけで作るよりデバッグしやすい
- GitHub PagesとlocalStorageだけの構成に合う

推奨レイヤー:

```text
背景レイヤー
地面・木・草レイヤー
作品レイヤー
生き物・植物アニメーションレイヤー
前景レイヤー
UIレイヤー
一時演出レイヤー
```

Canvasは補助的に使います。

| 用途 | Canvas使用 |
|---|---|
| まほうの粒子演出 | 使用可 |
| 一時的な光・きらきら | 使用可 |
| 作品サムネイル生成 | 必要なら使用可 |
| 森の主要データ表現 | 使用しない |
| 作品の永続保存 | 使用しない |

森全体を巨大なCanvasだけで作る設計は避けます。

## 12. 思い出の森

森は単なるゲーム背景ではなく、作品と思い出が残る場所です。

作品には次を関連付けます。

- 完成日
- 元のぬりえ
- 使用した色
- 今日達成したおしごと
- 親のひとこと
- お気に入り状態
- 森での配置場所

作品をタップすると、完成日、作品名、親のひとことを表示します。

## 13. 今日のぼうけん

1日の締めとして、Ver.1.0に「今日のぼうけん」画面を含めます。

表示内容:

- 今日の日付
- 今日達成したおしごと
- 今日獲得したスター
- 今日完成した作品
- 今日森に登場した生き物や植物
- 「きょうもがんばったね」のメッセージ
- 森へ進むボタン
- ホームへ戻るボタン

未達成のおしごとを責める表示はしません。連続記録が途切れたことも強調しません。

## 14. 親モード

親モードはVer.1.0では最小構成にします。

必要機能:

- 今日のおしごとの確認
- 完了状態の訂正
- 報酬スター数の変更
- おしごとの有効・無効
- 子どもの名前設定
- 今日の作品確認
- 親のひとこと入力
- JSONバックアップ
- JSON復元
- 全データ初期化
- バージョン表示

入り方:

1. ホーム右上の小さな保護者ボタンを押す
2. 2秒程度長押し
3. 確認画面を表示
4. 親モードへ進む

## 15. 4桁PINの扱い

Ver.1.0では、4桁PINは必須実装にしません。

理由:

- 家庭内の単一端末利用では長押しと確認で十分な可能性が高い
- PIN忘れ対応、再設定、初期化時の扱いが必要になり実装範囲が増える
- 子ども向けアプリの初期版では、親モード導線の分離を先に検証すべき

設計上は将来追加できるよう、`settings.parentGate` に次の概念を残します。

- `mode`: `hold_confirm` または `pin`
- `pinEnabled`
- `pinHash`
- `pinSalt`

Ver.1.0の推奨実装範囲は、`hold_confirm` のみです。PINはVer.1.1以降の候補とします。

## 16. データ互換方針

保存データには `schemaVersion` と `appVersion` を持たせます。起動時に `ensureDataShape()` 相当の補完処理を行い、欠けたプロパティを追加します。

補完処理の条件:

- 何度実行しても重複しない
- 既存のユーザー設定を上書きしない
- 初期おしごとはIDで存在確認して追加する
- 初期ぬりえもIDで存在確認して追加する
- マイグレーション前にバックアップを作る

- 生き物図鑑の本格実装
- 複数ワールド
- BGM
- 音声読み上げ
- PWA
- Service Worker
- 自由な森レイアウト編集
- 写真アップロード
- カメラ機能

拡張しやすくするため、次はデータ定義として扱います。

- おしごと
- ぬりえ

- 森の配置
- 解放条件
