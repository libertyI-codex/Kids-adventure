# Ver.1.0 試作20 実装メモ

## 目的
ホームを開いた直後に、星と仲間の状態を短時間で把握できる構成へ整理しました。おしごとは正式項目「おかたづけ」を追加し、親モードで家庭ごとの表示内容を設定できるようにしています。

## ホーム変更前後
変更前は大きな星カードと、下部ナビと重複する「おしごと・ぬりえ・せかい・さくひん」の4入口がホーム上部にありました。変更後は星を2つの小型表示へまとめ、重複入口を削除しています。下部ナビの4機能、たまご、お気に入りの鳥、とりのおうち、キッチンの入口は維持しています。

## 星表示
`profile.starTotals.spendableStars` と `profile.starTotals.lifetimeStars` を表示時に直接参照します。星の計算、台帳、消費処理は変更していません。2列の `home-star-strip` とし、390px幅と5桁の数値でも横スクロールが出ないようにしています。

## なかまのようす
ホーム初期描画に `renderCompanionStatus()` を組み込み、追加操作なしで表示します。

- activeたまごの残り成長ポイント
- お気に入り、または最初に取得した鳥の名前となかよしレベル
- `eggSystem.dailyActivity` から計算した今日のお世話数
- `lastBondMealDate` による今日の食事状況

計算値は重複保存しません。鳥・たまごがない場合は案内文だけを表示し、未定義値を画面へ出しません。

## jobSettings
`appData.jobSettings` にユーザー設定だけを保存し、正式定義を複製しません。

```json
{
  "dailyDisplayCount": 7,
  "enabledJobIds": [],
  "displayOrder": [],
  "customJobs": [],
  "dailySelectionsByDate": {}
}
```

### dailyDisplayCount
1〜10の整数へ正規化します。有効項目数が少ない場合は、有効項目をすべて表示します。

### enabledJobIds
重複・不明IDを除きます。0件になった場合は正式項目を1件だけ安全に有効化します。無効化しても履歴・星・たまごポイントは変更しません。

### displayOrder
有効・無効を問わず正式IDと有効なcustom IDを一意に保持します。不足IDは末尾へ追加し、不明IDを表示対象から除きます。

### customJobs
ID、名前、説明、プリセットiconKey、有効状態、作成・更新日時を保存します。名前は1〜20文字、説明は0〜60文字、最大20件です。名前変更でもIDは変更しません。HTMLは保存・解釈しません。

### dailySelectionsByDate
ローカル日付とjobIdのハッシュで候補順を決定し、同日の選択を保存します。再読み込みで引き直せず、最大60日分に制限します。

## 設定変更時の当日一覧
無効・削除済みIDを当日選択から外し、空いた枠だけを有効項目で補います。表示数を減らした場合は表示だけを縮小し、完了履歴を削除しません。並び順変更は選択済み集合を保ったまま表示順へ反映します。

## オリジナル項目
親モードの専用ダイアログで追加・編集します。アイコンは、おもちゃ箱、ほうき、お皿、洋服、本、植物、かばん、星の8種類です。標準項目は削除不可です。オリジナル削除後も、完了時に保存した表示名を履歴で使用できます。

## 標準設定リセット
標準項目の有効状態、正式順、標準表示数を戻します。オリジナル項目を残す操作と、強い再確認後にすべて削除する操作を分けています。達成履歴、スター、たまごポイント、報酬設定は再計算しません。

## データ移行とJSON互換
試作19以前のデータには標準設定を補完します。`ensureDataShape()` は設定の有効値を維持し、重複、不明ID、不正型だけを正規化します。JSONはappData全体を扱う既存方式のため、試作20のjobSettingsも書き出し・復元されます。

## safeStart
boot.js、safeStart、起動復旧画面は変更していません。通常初期化では不正jobSettingsを補正してホームとおしごとを描画します。保存データ全体の削除・初期化は行いません。
