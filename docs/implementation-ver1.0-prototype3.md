# Ver.1.0 試作3 実装メモ

## 目的

試作3では、毎日続けたくなるごほうび体験として「ふしぎなたまご」を追加し、ぬりえに「うま」を追加した。

## 追加した内容

- ぬりえテンプレート `coloring_horse_001`
- うまの領域: `mane`, `body`, `tail`, `legs`, `neck`, `eye`
- 解放順: 4, 8, 12, 16, 20, 24, 28スター
- たまご在庫 `eggInventory`
- たまご画面 `eggs`
- ホームのたまご導線
- 親モードの現在たまご数表示

## たまご仕様

累計スター10こごとに1個のたまごを付与する。

```json
{
  "id": "egg_stars_10",
  "createdAt": "2026-07-13T00:00:00+09:00",
  "earnedByStars": 10,
  "state": "new"
}
```

`state` は試作3では `new` のみ使用する。将来 `cracked`、`hatched` を追加できる。

## 互換性

`ensureDataShape()` は旧データに対して次を行う。

- `eggInventory` がなければ空配列を追加する
- 累計スターから不足しているたまごだけを追加する
- 既存たまごを削除しない
- 既に解放済みのぬりえを再ロックしない
- うまテンプレートを不足分として追加する

保存キーと `schemaVersion` は変更しない。

## 実装上の差分

- `js/eggs.js` を追加し、たまご同期を独立させた
- `js/stars.js` でスター増減後にたまご同期を実行する
- `js/migrations.js` で旧データ補完時にも同じ同期処理を実行する
- 森では `horse` を `trot` アニメーションで表示する

## 実装しないもの

- 孵化
- ひび演出
- たまご編集
- たまご削除
- たまごから新キャラクターを生成する処理
