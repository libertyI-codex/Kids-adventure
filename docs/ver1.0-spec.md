# Ver.1.0 仕様書

## 1. Ver.1.0の目的

Ver.1.0では、生活習慣と創作がつながる最小体験を完成させます。

必須体験:

```text
おしごとを完了する
↓
スターを獲得する
↓
ぬりえを解放する
↓
ぬりえを完成する
↓
まほうの仕上げを見る
↓
作品が森に登場する
↓
今日のぼうけんで振り返る
```

## 2. 実装範囲

Ver.1.0に含める画面:

1. ホーム
2. 今日のおしごと
3. スター獲得演出
4. ぬりえ一覧
5. ぬりえ編集
6. まほうの仕上げ
7. 森
8. 今日のぼうけん
9. 今日の作品・簡易アルバム
10. 親モード
11. データ管理

## 3. 実装対象外

Ver.1.0では次を実装しません。

- 複数プロフィール切り替え
- 春夏秋冬の自動切り替え

- 複数ワールド
- BGM
- 音声読み上げ
- 外部AI
- 外部API
- クラウド保存
- ログイン
- 端末間同期
- ランキング
- 課金
- 広告
- PWA
- Service Worker
- 自由な森レイアウト編集
- 写真アップロード
- カメラ機能
- index.html以外の複数HTML画面構成

## 4. ホーム

目的:

- 子どもが最初に見る入口
- 今日やることを迷わず始められる
- つかえるスターと森への導線を見せる

表示項目:

- 表示名: 結羽ちゃんの冒険
- 今日の日付
- つかえるほし
- これまでのほし
- 今日のおしごと進捗
- 今日完成した作品数
- 森の小さなプレビュー
- 保護者ボタン

ボタン:

- 「おしごと」
- 「ぬりえ」
- 「もり」
- 「さくひん」
- 保護者ボタン

正常系:

1. アプリ起動
2. 今日の日付を確認
3. 今日の記録がなければ作成
4. ホームを表示
5. 子どもが「おしごと」を押す

例外:

- 保存データがない場合は初期データを作成する
- 保存データが破損している場合は復元案内を表示する
- localStorage容量不足が疑われる場合は親モードで警告する

## 5. 今日のおしごと

目的:

- 今日のおしごとを完了し、スターを獲得する

初期おしごと:

| ID | 表示名 | 報酬 |
|---|---|---:|
| `task_brush_teeth` | はみがき | 1 |
| `task_get_dressed` | おきがえ | 1 |
| `task_picture_book` | えほん | 1 |
| `task_watering` | おみずあげ | 1 |

表示項目:

- おしごとカード
- おしごと名
- アイコン
- 報酬スター
- 完了状態
- 大きな「できた」ボタン
- 今日あつめたほし

操作:

- 未完了カードの「できた」を押す
- 完了済みカードは押せない、または押しても加算しない
- 完了済みは見た目を変える

正常系:

1. 子どもが未完了のおしごとの「できた」を押す
2. 今日の日別記録に `taskId` が未登録か確認
3. 未登録なら完了として保存
4. 累計スターとつかえるスターを加算
5. スター履歴に `earn_task` を追加
6. スター獲得演出へ進む

二重加算防止:

- 同じ `localDate` と `taskId` の組み合わせでは1回だけ加算する
- 画面連打対策として、押下直後にボタンを無効化する
- 保存処理後に再度日別記録を確認する

取り消し:

- 子ども画面には取り消しボタンを出さない
- 親モードからのみ訂正する
- 訂正時はスター履歴に `adjust_task_undo` を追加する

## 6. スター獲得演出

目的:

- 達成感を出す
- 次の行動へつなげる

表示項目:

- 大きなスター
- 「できたね」
- 獲得スター
- 今日の合計スター
- 次のおすすめボタン

ボタン:

- 「つぎのおしごと」
- 「ぬりえへ」
- 「きょうのぼうけん」

仕様:

- 演出は短くする
- スキップ可能
- 最後のおしごと完了後は「きょうのぼうけん」を目立たせる
- つかえるスターがぬりえ解放に足りる場合は「ぬりえへ」も表示する

## 7. ぬりえ一覧

目的:

- ぬりえを選ぶ
- つかえるスターで未解放ぬりえを解放する

初期ぬりえ:

| 解放順 | ID | 表示名 | 必要スター |
|---:|---|---|---:|
| 1 | `coloring_butterfly_001` | ちょうちょ | 1 |
| 2 | `coloring_flower_001` | おはな | 2 |
| 3 | `coloring_rabbit_001` | うさぎ | 3 |
| 4 | `coloring_cat_001` | ねこ | 4 |

表示項目:

- ぬりえカード
- 解放済み/未解放
- 必要スター
- つかえるほし
- 「ぬる」ボタン
- 「ほしでひらく」ボタン

解放条件:

- つかえるスターが必要スター以上
- まだ解放済みでない
- テンプレートが有効

正常系:

1. 子どもが未解放ぬりえの「ほしでひらく」を押す
2. 確認画面を表示する
3. つかえるスターを減算
4. 累計スターは減らさない
5. スター履歴に `spend_unlock_coloring` を追加
6. `unlocks.coloringTemplateIds` に追加
7. ぬりえ編集へ進める

例外:

- スター不足時は「もうすこしでひらくよ」と表示する
- 二重押し時は2回減算しない
- すでに解放済みの場合は減算せず編集へ進む

## 8. ぬりえ編集

目的:

- SVGぬりえに色を塗る

表示項目:

- SVGぬりえ
- 色パレット
- 選択中の色
- 「もどす」
- 「けす」
- 「できあがり」
- 「やめる」

色パレット:

- あか
- ぴんく
- きいろ
- おれんじ
- みどり
- みずいろ
- あお
- むらさき
- ちゃいろ
- しろ

操作:

- 色を選ぶ
- 塗り領域をタップする
- 「もどす」で直前の塗り操作を戻す
- 「けす」で選択領域を未塗りに戻す
- 1領域以上塗っていれば完成可能

保存:

- 下書きは `uiState` または `artworks` の draft として保存可能
- 完成時は `artworks` に正式保存
- Base64画像は大量保存しない

## 9. まほうの仕上げ

目的:

- 作品が冒険世界に入る体験を演出する

表示項目:

- 作品プレビュー
- まほう中メッセージ
- 進行アニメーション
- スキップボタン

文言例:

- 「いろをみているよ」
- 「もりにあうばしょをさがしているよ」
- 「まほうをかけているよ」
- 「できた！」

仕様:

- 5秒以内
- スキップ可能
- 実際のAIという表現は使わない
- 代表色、明るさ、色系統、塗り領域数を解析風に扱う
- 作品ID由来の固定シードで配置を決める

完了時:

- `artworks` に完成作品を保存
- `worlds.world_forest.placements` に配置を追加
- 今日の記録に完成作品IDを追加
- 森画面へ遷移

## 10. 森

目的:

- 完成作品と思い出を表示する

表示項目:

- 森背景
- 木・草・花
- 完成作品
- 今日追加された作品
- 作品詳細ポップアップ
- 今日のぼうけんへの導線

描画方式:

- SVG中心のレイヤー構造
- CSSアニメーションを併用
- Canvasは一時演出のみ補助使用

作品の動き:

- ちょうちょ: 小さく飛ぶ
- うさぎ: 小さく跳ねる
- おはな: 揺れる
- ねこ: 座る

タップ時表示:

- 作品名
- 完成日
- 使用色
- 親のひとこと
- お気に入り状態

自由配置:

- Ver.1.0では実装しない
- 自動配置のみ
- 配置座標は保存する

## 11. 今日のぼうけん

目的:

- その日の締めとして前向きに振り返る

表示項目:

- 今日の日付
- 今日達成したおしごと
- 今日獲得したスター
- 今日完成した作品
- 今日森に登場した生き物や植物
- 「きょうもがんばったね」
- 森へ進むボタン
- ホームへ戻るボタン

仕様:

- 未達成のおしごとは責めない
- 連続記録が途切れたことを強調しない
- 全おしごと完了後はこの画面への導線を強める

1日の全おしごと完了後の流れ:

```text
最後のおしごとを完了
↓
スター獲得演出
↓
きょうのぼうけんを表示
↓
つかえるスターがあればぬりえへ誘導
↓
森へ進む
```

## 12. 今日の作品・簡易アルバム

目的:

- 完成作品を見返す
- Ver.1.0では本格アルバムではなく簡易版にする

表示項目:

- 今日の作品
- 最近の作品
- 完成日
- 作品名
- お気に入り
- 親のひとこと

仕様:

- 大量のBase64サムネイル保存はしない

- 並び順は完成日の新しい順

## 13. 親モード

目的:

- 保護者が設定・訂正・データ管理を行う

入り方:

1. ホーム右上の保護者ボタンを2秒長押し
2. 確認画面を表示
3. 「おとながつかいます」を選ぶ
4. 親モードへ進む

Ver.1.0のPIN:

- 必須実装にしない
- 設計上は将来追加可能にする
- Ver.1.0では長押し + 確認を推奨

機能:

- 今日のおしごとの確認
- 完了状態の訂正
- 報酬スター数の変更
- おしごとの有効・無効
- 子どもの名前設定
- 今日の作品確認
- 親のひとこと入力
- JSONバックアップ
- JSON復元
- 全データ初期化
- バージョン表示

完了訂正:

- 完了を取り消す
- 日別記録に訂正履歴を残す
- スター履歴に訂正記録を残す
- つかえるスターが不足している場合は、0未満にしない
- 解放済みぬりえは自動で取り消さない

## 14. データ管理

目的:

- 長期利用のデータを守る

機能:

- JSONエクスポート
- JSONインポート
- インポート前の形式確認
- 一時バックアップ作成
- 全データ初期化
- バージョン表示
- localStorage容量警告

初期化:

- 1段階目: 初期化説明
- 2段階目: 確認チェック
- 3段階目: 最終確認
- 初期化前に可能ならバックアップを作る

## 15. スター計算

保持する値:

- `profile.starTotals.lifetimeStars`
- `profile.starTotals.spendableStars`
- `starLedger`

おしごと完了:

```text
lifetimeStars += reward
spendableStars += reward
starLedger += earn_task
```

ぬりえ解放:

```text
lifetimeStars 変更なし
spendableStars -= cost
starLedger += spend_unlock_coloring
```

親による完了取り消し:

```text
lifetimeStars -= reward
spendableStars -= min(spendableStars, reward)
starLedger += adjust_task_undo
```

訂正時にすでにスターを使っていた場合でも、解放済みぬりえは取り消しません。

## 16. 日付変更処理

日付判定は端末のローカル日付を使います。

日付キー形式:

```text
YYYY-MM-DD
```

処理タイミング:

- アプリ起動時
- ホーム表示時
- 今日のおしごと表示時
- ページ復帰時
- 日付が変わった可能性がある操作前

仕様:

- 日付が変わったら新しい `dailyRecord` を作る
- 前日の完了状態は残す
- 今日の完了状態だけ新しく空にする
- スター累計、つかえるスター、履歴は残す

## 17. 完了条件

Ver.1.0は次を満たしたら完了です。

- 初期データで起動できる
- 4つのおしごとを1日1回だけ完了できる
- スターが累計・つかえるに正しく加算される
- ぬりえ解放でつかえるスターだけ減る
- 4種類のぬりえを扱える
- ぬりえの領域カラーが保存される
- 完成作品が森に配置される
- 作品をタップして情報を見られる
- 今日のぼうけんで振り返れる
- 親モードで訂正とひとこと入力ができる
- JSONバックアップと復元ができる
- 実装対象外機能が混入していない
