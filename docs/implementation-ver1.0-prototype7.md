# こどもの冒険 Ver.1.0 試作7 実装メモ

## 目的

試作7では、試作6までの「思い出の森」を、複数の世界を選べる「せかい画面」へ拡張した。

追加した世界は次の5種類。

- 森
- 海
- 島
- 城
- 空島

既存の森、作品配置編集、保存データ、ぬりえ、スター、たまご、親モードは維持し、作品を好きな世界へ置けるようにした。

## 世界データ構造

正式な世界定義は `js/constants.js` の `WORLD_DEFINITIONS` に集約した。

各世界は次の情報を持つ。

- `id`
- `name`
- `title`
- `icon`
- `description`
- `theme`
- `unlocked`
- `unlockedAt`
- `designVersion`
- `displayOrder`
- `unlockCondition`

試作7では全世界を初期利用可能にしたため、`unlocked: true`、`unlockCondition: null` としている。

## 世界ID

使用する世界IDは次の通り。

| 世界 | ID | designVersion |
| --- | --- | --- |
| 森 | `world_forest` | `1` |
| 海 | `world_sea` | `1` |
| 島 | `world_island` | `1` |
| 城 | `world_castle` | `1` |
| 空島 | `world_sky_island` | `1` |

画面ルートは互換性を優先し、既存の `forest` ルートを継続使用している。データ上の正式な世界IDは `world_forest`。

## 各世界の背景設計

背景はすべてインラインSVGを中心に作成し、外部画像、外部SVG、Canvasによる画像生成は使っていない。

### 森

既存の森を維持し、空、太陽、木、丘、草地、池を配置した。既存作品の見た目と配置が大きく変わらないことを優先した。

### 海

明るい海中の背景にした。水面の光、泡、砂地、岩、海藻、サンゴ風装飾を配置し、イルカが自然に見える構成にした。

### 島

青空、海、砂浜、草地、丘、ヤシの木、小道を配置した。うさぎ、馬、恐竜、花などを置いても不自然になりにくい構成にした。

### 城

怖くない明るい中庭にした。城壁、塔、旗、石畳、芝生、花壇、噴水を配置し、ねこや馬に合う世界にした。

### 空島

雲の上に浮かぶ世界として、青空、雲、浮島、小さな滝、虹、光の粒を配置した。ちょうちょが飛んでも自然に見える構成にした。

## 既存森データの移行

試作6以前の既存作品には所属世界がないため、`ensureDataShape()` で配置データへ `worldId: "world_forest"` を補完する。

維持する情報。

- `xPercent`
- `yPercent`
- `zIndex`
- `isManual`
- `templateId`
- `regionColors`
- 完成日
- 親コメント
- お気に入り
- 固定seed

不正な `worldId` がある場合も作品は削除せず、`world_forest` へ安全に戻す。

## 作品worldIdの保存場所

所属世界は `placement.worldId` に保存する。

1つの作品は同時に1つの世界だけに所属する。世界間移動時は移動元の `placements` から削除し、移動先の `placements` へ追加する。作品データ本体は削除しない。

## 新規作品の配置先選択

まほうの仕上げ完了後、「どのせかいに おく？」を表示し、5世界から選択できるようにした。

おすすめ世界は案内のみで、どの作品もどの世界にも置ける。

選択せずに閉じた場合でも、作成時に推奨世界へ自動配置済みのため作品は失われない。

推奨世界は次の通り。

- ちょうちょ: `world_forest`
- おはな: `world_forest`
- うさぎ: `world_forest`
- ねこ: `world_castle`
- イルカ: `world_sea`
- きょうりゅう: `world_island`
- うま: `world_island`

## 世界間移動

せかい画面に「さくひんをうつす」ボタンを追加した。

操作の流れ。

1. 現在の世界にいる作品を選択
2. 移動先世界を選択
3. 確認
4. 移動
5. 移動先世界を表示

移動後は移動先世界の自動配置を再計算し、`isManual: false` にする。移動元世界の座標はそのまま流用しない。

## 世界別の自動配置

`js/worlds.js` に世界別の配置スロットを定義した。

配置候補は世界ごとに分け、同じ座標だけに偏らないようにした。

- 森: 草地、池の近く、木の間、空側
- 海: 海底、サンゴの近く、水中中央、水面寄り
- 島: 砂浜、草地、丘、木の近く
- 城: 中庭、花壇、城門前、芝生
- 空島: 浮島の草地、雲の近く、空側、虹の近く

作品IDと世界IDから固定seedを作り、同じ作品を同じ世界に置いた場合は同じ自動配置になりやすい設計にしている。

## 世界別の自由配置

試作6の「ならべかえ」をすべての世界へ拡張した。

編集対象は現在開いている世界の作品だけ。他の世界の作品は表示も移動もしない。

保存する座標。

- `xPercent`
- `yPercent`
- `zIndex`
- `isManual`
- `updatedAt`

座標は各世界の描画エリアを基準にするため、画面全体や下部ナビは基準にしない。

## zIndex処理

ドラッグ中の作品は一時的に前面へ出す。確定時は `normalizeZIndexes()` で現在世界内のzIndexを1から順番に正規化し、極端に大きな値が増え続けないようにした。

世界間移動後も移動先世界内でzIndexを正規化する。

## UI状態

最後に開いていた世界は `uiState.selectedWorldId` に保存する。

無効なIDが保存されている場合は `world_forest` へ戻す。配置編集モード自体は再起動後まで維持しない。

## 互換性対応

維持したもの。

- 既存作品
- 既存の森配置
- `regionColors`
- `designVersion`
- 解放済みぬりえ
- スター履歴
- たまご
- 親コメント
- お気に入り
- JSONバックアップ
- BGM設定
- 効果音設定
- 親モード
- 保存キー
- schemaVersion

`ensureDataShape()` は、5世界の不足分だけを追加し、複数回実行しても世界や配置を重複させない。

## iPhone Safari対応

親モード長押しと配置編集は、試作6までのpointerイベント方針を維持した。

配置編集モード中の作品だけに次を適用する。

- `user-select: none`
- `-webkit-user-select: none`
- `-webkit-touch-callout: none`
- `touch-action: none`

通常モードではドラッグを禁止し、作品タップで詳細を開く。配置編集モードでは詳細モーダルを開かない。

## 検証結果

実装後に確認した内容。

- 5世界が1回だけ追加される
- `ensureDataShape()` を複数回実行しても重複しない
- 既存 `world_forest` が維持される
- 既存作品に `world_forest` が補完される
- 不正な `worldId` は `world_forest` へ戻る
- `uiState.selectedWorldId` が保存・復元される
- 無効な `selectedWorldId` は `world_forest` へ戻る
- 新規作品は推奨世界へ配置される
- まほうの仕上げ後に配置先を選べる
- 世界間移動で移動元から消え、移動先に表示される
- 世界ごとの自由配置で `xPercent` / `yPercent` が保存される
- 現在世界だけを自動配置へ戻せる
- zIndexが世界ごとに正規化される
- JS構文、読み込み順、外部依存なしを確認
- 保存キーとschemaVersionを変更していない

## 手動確認項目

ブラウザ操作が必要な確認。

- iPhone Safariで世界選択モーダルが見切れないこと
- 各世界の背景が390px前後で横スクロールしないこと
- 背景装飾をドラッグできないこと
- 5世界すべてで作品を指で並べ替えられること
- ドラッグ終了直後に作品詳細が誤表示されないこと
- BGMと効果音のON/OFFが従来通り動作すること
- JSONバックアップと復元を実ブラウザで実行できること
